The Neural Pipeline

How we convert a static document into a dynamic conversation.

1

Ingestion & Extraction

User uploads a PDF. PyMuPDF extracts raw text, handling complex layouts.

2

RAG Context Synthesis

LangChain and an ephemeral ChromaDB chunk and embed the text using local HuggingFace embeddings. We retrieve only the most relevant context to prevent AI hallucinations.

3

LLM Script Orchestration

Llama 3.1 (via OpenRouter) acts as the scriptwriter, crafting a 2-person dialogue based exclusively on the retrieved facts.

4

Neural Speech Synthesis

Dialogue lines are processed concurrently using Microsoft Edge TTS for lifelike intonation.

5

Audio Mastering

Pydub and FFmpeg stitch the audio clips together with natural pauses into a final MP3.

Backend ↔ Frontend Architecture

1. REST API Handshake

User uploads PDF. Backend validates & hands heavy processing to a background thread. Frontend gets Job ID instantly to stay responsive.

2. Server-Sent Events (SSE)

Persistent connection PUSHES progress (e.g., 'Synthesizing voices… 80%'). Live logs stream to the Neural Terminal UI without polling.

3. Unified Hosting

FastAPI serves both API + Frontend (HTML/CSS/JS) on a single port. No CORS issues, simple one-command deployment.

Tech Stack & Jargon

⚡

FastAPI

High-speed Python web framework driving the backend logic and Server-Sent Events (SSE) for real-time UI updates.

🧠

RAG

Retrieval-Augmented Generation. Forces the AI to read your specific document before speaking, ensuring absolute factual accuracy.

🗄️

Vector Database

We use ChromaDB to store text as numbers (embeddings), allowing us to search the PDF by meaning rather than exact keywords.

🎙️

Neural TTS

Advanced AI models that don't just read text, but understand context to add breaths, pauses, and emotion.

Challenges & Key Learnings

Challenges Faced

  • Real-time progress streaming with SSE required careful async handling in FastAPI.
  • PDF layouts with columns & tables needed custom extraction logic via PyMuPDF.
  • Prompt engineering to maintain consistent persona voices across long dialogues.
  • Concurrently generating & ordering audio clips without desync in Pydub.

Key Learnings

  • RAG is not just for chatbots — it's a core accuracy layer for any generative pipeline.
  • SSE is a superior alternative to polling for real-time UX — simpler and more efficient.
  • Neural TTS (edge-tts) can rival paid APIs with the right voice config & pausing.
  • Full-stack AI apps need end-to-end thinking — from chunking strategy to audio format.
